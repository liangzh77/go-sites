# 演示管理接口

`cmd/demo-server` 提供登录后可用的演示上传、删除、禁用能力。

## 路由

- `POST /api/demo/session`：提交 `{ "password": "..." }`，成功后设置 HttpOnly cookie。
- `GET /api/demos`：列出演示。已登录时包含禁用项，未登录时只返回启用项。
- `POST /api/demos`：multipart 上传，字段为 `file` 和可选 `title`。支持 `.html`、`.md`、`.zip`。
- `PATCH /api/demos/{slug}`：提交 `{ "disabled": true | false }` 或 `{ "title": "新名称" }`。改名会同步更新 slug、地址和演示文件夹路径。
- `DELETE /api/demos/{slug}`：删除演示记录和对应文件。
- `GET /demo/{slug}/...`：由后端服务演示文件；禁用项返回 404。

## 运行期数据

生产环境建议设置：

```text
DEMO_DATA_DIR=/srv/apps/go-sites-demo/shared
SITE_ROOT=/srv/sites/liangz77.cn/current
DEMO_ADMIN_PASSWORD=<server secret>
PUBLIC_ORIGIN=https://liangz77.cn
```

上传内容会保存到 `DEMO_DATA_DIR/demos/{slug}`，清单保存到 `DEMO_DATA_DIR/manifest.json`。
